Use case diagram

Use case #1

Use case name

Visualize Home Page

Actors
  • User

Description

The player visualizes the home page GUI of the game where it is possible to go to the level selection menu, resume a game (if one is saved), go to the github page and go to the documentation page

Preconditions

None 

Flow
  • The player starts the application
  • The player clicks on the “Home” button on the top bar on either scene #2 or scene #3

Postconditions

The player is viewing the home page GUI

Use case #2

Use case name

Go to level selection men√π

  • User

Description

The player visualizes the level selection menu GUI where it is possible to select a level and start a new game.

Preconditions

None 

Flow
  • The player clicks on the “new game” on the Home menu
  • The player clicks on the “Go Back” button on the top bar on scene #3

Postconditions

The player is viewing the level selection GUI

Use case #3

Use case name

Start a new game

Actors
  • User

Description

The player starts a new game.

Preconditions

Use case #2 

Flow
    1. Performing use case #1
    2. The player clicks on the “new game” button
    3. Performing use case #2
    4. The player selects a level clicking on one of the configurations images
    1. Player is in scene #3
    2. The player clicks on the “Go Back” scene #3
    3. Performing use case #2
    4. The player selects a level clicking on one of the configurations images

Postconditions

The player is on scene #3 and can start playing from one of the default configurations

Use case #4

Use case name

Solve a level

Actors
  • User

Description

The player can move the blocks according to klotski rules and solve a puzzle

Preconditions

User must have performed use case #3 or use case #8

Flow
  1. The player is on scene #3 after performing use case #3 or use case #8
  2. The player moves the blocks according to the klotski rules
  3. If final configuration ir reached the player sees "Level Completed"

Postconditions

"Level Completed" message appears if final configuration is reached

Use case #5

Use case name

Perform Best Move

Actors
  • User

Description

The player clicks on the "Next Best Move" button on scene #3 to perform the best move on board

Preconditions
  • Player must be on scene #3
  • The configuration on board must be the starting configuration or player must have performed all the best moves until clicking on the "Next Best Move" button

Flow
  • The player clicks the "Next Best Move" button on scene 3

Postconditions

The player sees the automatic performing of the next best move

Use case #6

Use case name

  • User

Description

Preconditions

  • At least one move must have been performed

Flow
  • The player clicks on the "Undo" button on scene #3

Postconditions

The board configuration return as it was before the latest move was performed
