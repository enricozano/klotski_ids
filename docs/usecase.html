Use cases

The first design meetings tackled about the actions the user would have to be able to perform. The team decided to work on an application with three main scenes:

  • Scene #1: The starting menù
  • Scene #2: The level selection menù
  • Scene #3: The main playing menù
Later, a set of use cases were developed for each scene.
The team so proceded by creating the use case diagram.

Use case diagram

Use case #1

Use case name

Start a game

Actors
  • Player

Description

The player starts a game, either performing use case #2 or use case #3.

Preconditions

None 

Flow
  1. The player is on Scene #1
    • The player performs use case #2
    • The player performs use case #3

Postconditions

The player is on scene #3 viewing the game board with the chosen configuration.

Use case #2

Use case name

Start a new game

Actors
  • Player

Description

The player starts a new game.

Preconditions

None 

Flow
    1. The player clicks on the “New Game” on scene #1
    2. The player performs use case #4
    1. The player clicks on the “Back” on the top bar in scene #3
    2. The player performs use case #4

Postconditions

The player is on scene #3 viewing the game board with one of the default configurations.

Use case #3

Use case name

Resume Game

Actors
  • Player

Description

The player resume the saved game.

Preconditions

Use case #10 

Flow
  • The player clicks on the “Resume Game” on Scene #1

Postconditions

The player is on scene #3 viewing the game board with the previously saved configuration.

Use case #4

Use case name

Choose initial configuration

Actors
  • Player

Description

The player choose the initial configuration from four default configurations.

Preconditions

None

Flow
  1. The player is on Scene #2
  2. The player clicks on one of the default configurations

Postconditions

The player is on scene #3 viewing the game board with the chosen default configuration.

Use case #5

Use case name

Play the game

Actors
  • Player

Description

The player can play the game according to klotski rules and solve a puzzle.

Preconditions

Use case #1

Flow
  1. After performing use case #1 the player can start to play the game according to the klotski rules and use the other funcionality described in the use cases #8, #9 and #10

Postconditions

"Level Completed" message appears if final configuration is reached

Use case #6

Use case name

Move the blocks

Actors
  • User

Description

The player can move the blocks according to the klotksi rules.

Preconditions

Use case #1

Flow
  • The player drag a block in the board according to the klotski rules.

Postconditions

The board configuration modifies according to the moved blocks

Use case #7

Use case name

Undo last move

Actors
  • Player

Description

The board configuration return as it was before the latest move was performed.

Preconditions

Use case #6 must have been performed at least once

Flow
  • The player clicks on the “Undo” button on Scene #3.

Postconditions

The board configuration must return as it was before the latest move was performed.

Use case #8

Use case name

Reset

Actors
  • Player

Description

The player reset the board configuration to the default one.

Preconditions

Use case #6 must have been performed at least once

Flow
  • The player clicks on the “Reset” button on Scene #3

Postconditions

The board configuration returns to the default one chosen

Use case #9

Use case name

Next Best Move

Actors
  • User

Description

The player makes the system playing the next best move.

Preconditions

Flow

Postconditions

The next best move must be performed

Use case #10

Use case name
